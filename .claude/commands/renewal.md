# /renewal — サイトデータ最新化スキル

記事以外のサイト内データ（経済指標・政治ニュース・議席数・内閣・法案状況など）を最新情報に更新するスキルです。

## ワークフロー

### 1. 現在のデータ状態を確認
以下のファイルの `lastUpdated` や最新日付を確認し、更新が必要な範囲を特定する：

| ファイル | データ内容 |
|---|---|
| `src/lib/economy.ts` | 経済指標（GDP・CPI・失業率・日経平均・為替・金利等）、経済ニュース |
| `src/lib/politicsNews.ts` | 政治ニュースタイムライン |
| `src/lib/politics.ts` | 議席数・内閣閣僚・議員一覧 |
| `src/lib/partyInfo.ts` | 政党情報（党首・政策） |
| `src/lib/cabinetPolicy.ts` | 内閣の政策目標・進捗率 |
| `src/lib/debates.ts` | 審議中法案の状況・タイムライン |
| `src/lib/passed.ts` | 可決済み法案 |
| `src/lib/enacted.ts` | 施行済み法案 |
| `src/lib/industryFinancials.ts` | 業界財務データ |

### 2. WebSearchで最新情報を収集
**並列でエージェントを起動し、以下を検索する：**

#### 経済指標
- 日経平均株価（終値）
- USD/JPY為替レート
- CPI（消費者物価指数）の最新公表値
- 完全失業率の最新公表値
- 日銀政策金利（変更があれば）
- 10年国債利回り
- WTI原油価格
- 経常収支・貿易収支

#### 政治
- 国会の最新動向（法案の審議・可決・否決）
- 内閣の人事異動
- 衆参の議席数変動（補欠選挙等）
- 主要な政治ニュース

#### 法案
- 審議中法案のステータス変更
- 新たに可決・施行された法律

### 3. データファイルを更新

#### 経済指標 (`economy.ts`)
- `value` を最新の数値に更新
- `lastUpdated` を更新
- `change` を前回値に更新
- `sparkline` 配列を最新データでスライド（古い値を削除、新しい値を追加）
- `description` を最新の市況に更新
- `trend` と `status` を適切に設定

#### 経済ニュース (`economy.ts` の `economyNews`)
- 新しいニュースを配列の先頭に追加
- `date`, `title`, `summary`, `category`, `impact`, `relatedIndicators` を設定

#### 政治ニュース (`politicsNews.ts`)
- 新しいニュースを配列の先頭に追加
- `date`, `time`, `title`, `summary`, `tag`, `tagColor` を設定
- tagColor: 外交=#2b6cb0, 政策=#d69e2e, 施行=#805ad5, 法案=#38a169, 予算=#e53e3e, 国会=#e53e3e

#### 法案データ (`debates.ts`, `passed.ts`, `enacted.ts`)
- ステータスが変わった法案を移動（審議中→可決、可決→施行）
- タイムラインに新しいイベントを追加

#### 議席数・内閣 (`politics.ts`)
- 補欠選挙や会派変更があれば議席数を更新
- 内閣改造があれば閣僚を更新

### 4. ビルド確認
```
npx next build
```
エラーがなければコミット・プッシュ。

### 5. 変更不要の判断
検索の結果、前回更新時点から変更がないデータは更新しない。
変更があったもの・なかったものの両方を最後にユーザーに報告する。

## 必須ルール

- **WebSearch/WebFetchで一次情報を必ず確認する**（訓練データに頼らない）
- 数値は公式ソース（統計局・日銀・財務省・IMF等）で裏取りする
- 記事データ（`src/lib/ict.ts` 等の articles 配列）は変更しない
- 既存のインターフェース・型定義は変更しない
- データの時点を明確にする（`lastUpdated` フィールド）

## 使い方

```
/renewal
```

引数なしで実行。現在日時点の最新データに自動更新します。
