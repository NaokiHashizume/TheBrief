# /article — TheBrief記事作成スキル

TheBriefプロジェクトの記事を作成・追加するためのスキルです。

## 必須ルール

### 1. 一次情報を必ず当たる
- **WebSearch・WebFetchを使って、公式ソースの一次情報を必ず確認すること**
- 官公庁（省庁・国会・内閣法制局等）、企業のIR・プレスリリース、公式統計データを優先
- ニュース記事だけに頼らず、元データ・原典を辿って正確性を担保する
- 数値データ（売上高・利益率・株価・統計数値等）は必ず公式ソースで裏取りする
- 日付・人名・法案名等の固有名詞は特に正確性を重視する

### 2. 最新情報の確認
- 記事のテーマに関する最新の動向をWebSearchで必ず調査する
- 古い情報や訓練データに頼らず、現時点での最新状態を反映する
- 「2026年4月現在」等、データの時点を明示する

### 3. 記事データの構造
- `src/lib/` にデータファイル（TypeScript）を作成
- 既存のインターフェース（`IndustryArticle`, `TradingArticle`等）に準拠
- セクションごとに `diagramId` を設定し、対応する図解コンポーネントを作成

### 4. 図解コンポーネント
- `src/components/` に `"use client"` のReactコンポーネントとして作成
- Tailwind CSSで統一されたデザイン（brief-border, brief-card等のクラスを使用）
- データの視覚化を重視（比較表、タイムライン、構成図等）

### 5. ページの作成
- 動的ルート `[slug]/page.tsx` で記事詳細ページを実装
- `generateStaticParams` でSSG対応
- パンくずリスト・メタデータ・JSON-LDを必ず含める

### 6. Git運用
- 新しいブランチで作業する（例: `article/テーマ名`）
- ビルド確認後にコミット・プッシュ
- デスクトップアプリのClaude Codeがマージを担当

## 使い方

```
/article [カテゴリ] [テーマの説明]
```

例:
- `/article industry/ict AIモデル比較記事`
- `/article politics 最新の法案一覧`
