GitHubスター数33.5万。公開4ヶ月。Reactが10年かけた記録を60日で抜いた。

OpenClawは2026年を象徴するオープンソースプロジェクトだ。

オーストリアの開発者が「1時間で作ったプロトタイプ」から始まり、創業者はOpenAIに引き抜かれ、172のスタートアップが上に構築されている。

以下、技術・使い方・実践Tips・セキュリティまで徹底的にまとめる。

■ OpenClawとは何か

一言で言うと「ローカルで動く自律型AIエージェント」。

・WhatsApp/Telegram/Discordからチャットで指示
・PCのファイル操作、ターミナル実行、ブラウザ制御を自動化
・メール送受信、カレンダー管理まで対応
・無料・オープンソース（MIT License）

映画のJARVISに最も近いAI、と言われている。

ChatGPTやClaudeが「質問に答えるAI」なら、
OpenClawは「実際にPCを操作してタスクを完了するAI」。

ここが決定的に違う。

■ 技術アーキテクチャ（開発者向け）

3つのコア要素:

【1. ローカルゲートウェイ】
Node.jsアプリがPC上で動作。チャットアプリからの入力をLLMに送信→返ってきた指示でローカルツールを操作する。

【2. BYOM（Bring Your Own Model）】
50以上のLLMプロバイダーに対応:
・Claude Opus 4.6
・GPT-5.4
・Gemini 3.1
・DeepSeek
・Ollama（ローカルLLM）

モデル参照: provider/model形式
例: opencode/claude-opus-4-6

→ 簡単なタスクは安いモデル、複雑な推論は高性能モデルに自動ルーティング可能。

Ollama + OpenClaw = 完全無料・オフライン運用が実現。APIキー不要。

【3. スキルシステム（ClawHub）】
・SKILL.md + 設定ファイルのバージョン管理されたバンドル
・2月初旬5,700 → 3月末13,729スキルに急拡大
・ベクトル検索でスキル発見
・CLI: install（バージョン固定可）/ search / publish

■ セットアップ方法（15分で完了）

【必要環境】
・Node.js 24（推奨）or 22 LTS
・APIキー（Anthropic/OpenAI/Google）※ローカルLLMなら不要
・macOS / Linux / Windows（WSL2）

【3つのインストール方法】

① Docker（最速・5分）
② npm CLI（15分）
③ clawoneclick.com（60秒・設定ゼロ）

【CLI手順】
1. Node.js 24をインストール
2. npm install openclaw
3. openclaw onboard --install-daemon
4. ガイドに従いゲートウェイ・モデル・チャンネルを設定

【Windows限定】
PowerShellで1コマンド:
iwr -useb https://openclaw.ai/install.ps1 | iex

■ 実践ユースケースBEST 10

① 朝のデイリーブリーフィング（最人気）
天気・カレンダー・タスク・ニュース・健康データを1メッセージに集約。5個のアプリを開く代わりにOpenClawが1つにまとめる。

② メール自動化（キラーアプリ）
受信→分類→優先度判定→下書き作成→定型返信の自動送信。週数時間の工数削減。

③ 会議の文字起こし→タスク自動生成
録音→文字起こし→決定事項・担当者・期限を抽出→Jira/Linearに自動作成。担当者アサインまで。

④ クライアントオンボーディング
新規受注→フォルダ作成→歓迎メール→キックオフ会議設定→フォローアップリマインダーを一括自動。

⑤ 音声メモの自動日記化
1日の音声メモを文字起こし→毎晩きれいな日記エントリーに自動変換。

⑥ 荷物追跡
注文メールから追跡番号抽出→配送ステータス管理→「配達中」「遅延」アラート。

⑦ SNSブランドモニタリング
定期検索→センチメント分析→影響力の大きいアカウント特定→要対応投稿をハイライト。

⑧ スマホからリモートコーディング
WhatsApp経由でPCのターミナルを操作。移動中にコードを実行。

⑨ リサーチ→ドキュメント自動生成
Web調査→構造化ドキュメント作成→Notionに自動同期。

⑩ AI音声通話
イベントのゲストリストに1人ずつ自動電話→出欠確認→回答まとめを生成。

■ 他ツールとの比較と使い分け

OpenClaw = 汎用AIエージェント（生活全般）
Claude Code = コーディング特化エージェント（ターミナル内）
Cursor = IDE内蔵AIアシスタント（エディタ内）

【おすすめスタック】
・Cursor → 日常コーディング
・Claude Code → 自律タスク、セキュリティ重視の開発
・OpenClaw → 非コーディングの自動化全般

3つは競合ではなく補完関係にある。

■ セキュリティリスク（ここが超重要）

正直に言う。OpenClawにはセキュリティ上の深刻なリスクがある。

【報告済み脆弱性】
・CVE-2026-25253（CVSS 8.8）: リソース転送の不備
・CVE-2026-24763: コマンドインジェクション
・CVE-2026-26322: SSRF
・CVE-2026-26329: パストラバーサル
・CVE-2026-30741: プロンプトインジェクション経由のコード実行

【ClawHubの闇】
セキュリティ監査でスキルの最大12%が悪意あるものと判明。認証情報の窃取やプロンプトインジェクションのペイロードが含まれていた。

Cisco: 「スキルリポジトリに十分な審査がない」
中国当局: 国有企業・政府機関でのOpenClaw使用を禁止

【公開インスタンスの問題】
1月末の1週間で公開インスタンスが1,000→21,000以上に急増。
42,665の公開インスタンスのうち5,194が脆弱。

【必須セキュリティ対策】
✓ Canvas Hostのバインドを127.0.0.1に変更
✓ 業務用/個人アカウントでは実行しない
✓ 機密データのあるデバイスで動かさない
✓ ClawHubスキルは信頼できるもののみ
✓ 定期的にバージョンアップデート
